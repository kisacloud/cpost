# Project CPOST: AI Multi-Sector Report Generator

## 1. 프로젝트 개요
이 프로젝트는 Gemini 1.5 Pro를 활용하여 정책, 기술, 산업 등 다양한 분야의 최신 정보를 수집하고 종합 리포트를 작성하여 Slack으로 전송하는 자동화 시스템입니다.

## 2. 핵심 설계 원칙 (Strict Rules)
- **설정 분리**: 모든 API 키, 환경 변수, 고정 경로는 `config/config.py`에서 관리한다.
- **프롬프트 자산화**: 프롬프트는 로직과 섞이지 않도록 `prompts/` 폴더 내에 분야별로 독립된 모듈로 관리한다.
- **모듈화**: 정보 수집(Collector), 분석(Engine), 전송(Utils) 로직을 엄격히 분리하여 향후 특정 분야(정책/기술 등)만 추가하거나 수정하기 용이하게 한다.

## 3. 코드 작성 가이드라인
- **Python 버전**: 3.12.x 기반으로 작성한다.
- **타입 힌트**: 모든 함수 선언 시 Type Hint를 사용하여 가독성과 안정성을 높인다.
- **비동기 고려**: 향후 대량의 정보 수집을 위해 `httpx`나 `asyncio`를 고려한 구조를 지향한다. (단, 실행머신의 한계(1G 메모리의 마이크로서버)를 감안하여 단계적으로 효율적인 접근 추진)

## 4. 향후 확장 로드맵 (에이전트 참고용)
1. **분야별 수집 분화**: 현재 통합 수집 방식에서 정책(Policy), 기술(Tech), 산업(Industry) 키워드를 분리하여 병렬 수집할 예정.
2. **종합 로직**: 각 분야별로 생성된 중간 리포트를 다시 한번 종합하여 '인사이트 요약'을 만드는 상위 에이전트 로직 추가 예정.
3. **스케줄링**: `scheduler.py`를 통해 분야별로 리포트 생성 주기(매일/매주)를 다르게 가져가는 기능을 구현할 예정.
